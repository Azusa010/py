# 错误和异常

Python是一门解释型语言，只有在程序运行后才会执行语法检查，所以只有在运行或测试程序的时候才能检查程序是否能够正常运行

### 语法错误

解释器不能正确解释程序

### 异常

 解释器能正常解释程序，但是运行时报错



## 异常处理

### try expect

如果发生了异常，会忽略`try`中剩余代码，执行`except`代码

如果没有异常，会忽略`except`代码，继续向下执行

```py
try:
    可能发生异常的代码
except:
    异常处理的代码
```

如:

```py
try:
    res = 10/0
    print(res)
except:
    print('an error occurred') #an error occurred
```

### 对不同类型的异常的处理

在程序执行时，如果发生异常，会创建一个异常类型的对象

`except`可以捕获异常,这可以在`except`后指定捕获异常的类型



```py
try:
    可能发生异常的代码
except 异常类型 as 变量名:
    异常处理的代码
except (异常类型1,异常类型2) as 变量名1:
    异常处理的代码
except:
    异常处理的代码
```

如下:

```py
try:
    res = 10/0
    print(a)
except  ZeroDivisionError as e:
    print('除数不能为零')
except NameError as e :
    print('变量未定义')
except:
    print('未知错误')
```

### else

可以将else放在expect之后。如果try中代码没有异常，将执行else中的代码

```py
try:
    res = 10/1
except  ZeroDivisionError as e:
    print('除数不能为零')
else:
    print('没有异常发生') #没有异常发生
```

### finally

不管是否发生异常，都会执行finally中的代码

大部分场景，把代码放到finally和try-except块外的效果是一样的

- 放到try-except块外的代码只会在try-expect结构正常执行完后才会执行
  
  - 如果try块中出现异常，并且没有被except捕获或者except中出现了新的异常导致`程序终止`，那这部分代码不会被执行

一般把资源释放代码放到finally中

```py
try:
    res = 10/1
except  ZeroDivisionError:
    print('除数不能为零')
else:
    print('没有异常发生')
finally:
    print('无论是否发生异常都会执行')
```

## 抛出异常

### raise

**主动抛出异常**，手动触发错误，中断正常代码流程，配合 `try-except` 捕获处理。

```py
#语法
raise 异常类型("异常描述“)
```

```py
def add(num1,num2):
    if isinstance(num1, int) and isinstance(num2, int):
        return num1 + num2
    else:
        raise TypeError('参数类型错误')  # 手动抛出异常
print(add(1, 2))
try:
    add(1, '2')
except TypeError as e:
    print(e)
```

### assert 断言

`assert`用与判断一个表达式，在表达式条件为False除法异常

一般用于`调试程序`

```py
assert 表达式[,异常描述]
```

```py
def add(num1,num2):
    assert isinstance(num1,int) and isinstance(num2,int), '参数类型错误' 
    return num1 + num2
try:
    add(1, '2')
except AssertionError as e:
    print(e)

```

### 自定义异常

通过直接或间接继承Exception类来创建自己的异常

```py
class MyError(Exception):
    def __init__(self, message):
        self.message = message
    def __str__(self):
        return super().__str__()

def add(num1,num2):
    if isinstance(num1, int) and isinstance(num2, int):
        return num1 + num2
    else:
        raise MyError('参数类型错误')  # 手动抛出异常
try:
    add(1, '2')
except MyError as e:
    print(e)
```

### 异常的传递

当存在try嵌套或函数嵌套时，若内层出现了异常且在内层无法处理，就会一层一层的向外传递，直到异常被处理或程序报错

try的嵌套

```py
try:
    try:
        try:
            res = 10/1
        except ZeroDivisionError as e:
            print('第三层', e)
    except TypeError as e:
        print('第二层', e)
except Exception as e:
    print('第一层', e)
```

函数嵌套

沿着函数的调用顺序反向传递

m1()->m2()-m3()

```py
def m3():
    print(1/0)
def m2():
    m3()
def m1():
    m2()
m1()
```

- `m3` 内部发生 `1/0`，产生 `ZeroDivisionError`
- `m3` 没有捕获异常，异常抛给调用它的 `m2`
- `m2` 无异常处理，异常继续抛给调用它的 `m1`
- `m1` 无异常处理，最终抛到程序顶层，程序崩溃并打印完整报错栈



## `with`关键字

`with`语句用于异常处理，封装了`try catch finally`编码范式

无论代码块正常结束还是抛出异常，都会自动执行资源清理操作，避免忘记释放资源导致泄漏。

### 语法

```py
with 表达式 as 接收变量:
    #代码块 
```

表达式：`with 表达式` 执行后，**返回值必须是一个`上下文管理器对象`**，该对象要实现`__enter__` 和 `__exit__`方法

接收变量：可选，用于存储表达式的`__enter__()`的返回值

```py
try:
    with open("test.txt", "w") as f:
        f.write(str1)
finally:
    print(f.closed) #True
```
