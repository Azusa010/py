# 进程与线程

## 并发与并行

### 并发

单个CPU处理多个任务。各个任务交替执行一段时间

### 并行

多个CPU同时执行多个任务

## 多进程

进程是操作系统进行资源分配的基本单位

每个进程都有一块独立的内存空间

多进程是指在操作系统中同时运行多个程序

### 使用`multiprocessing.Process`创建进程

```py
multiprocessing.Process(group=None,target=None,name=None,args=(),kwargs={},daemon=None)
```

`group`:应当始终为None，它的存在是为了与`threading.Thread`兼容

`target`:由`run()`方法发起调用的可调用对象，默认为None

`name`:进程名称，默认为None则自动分配

`args`:针对目标调用的参数元组

`kwargs`:针对目标调用的关键字参数字典

`daemon`:是否为守护进程,True或者False,默认为None则继承父进程

```py
def write_file():
    """像文件中写数据"""
    with open('test.txt','a') as f:
        while True:
            f.write('hello world\n')
            f.flush() #手动将缓冲区的数据刷写到文件
            time.sleep(0.5)     

def read_file():
    """从文件中读数据"""
    with open('test.txt','r') as f:
        while True:
            time.sleep(0.5)
            print(f.readline())

if __name__ == '__main__':
    #避免子进程再创建子进程
    p1 = multiprocessing.Process(target=write_file)
    p2 = multiprocessing.Process(target=read_file)

    p1.start()
    p2.start()
```

### 通过进程池创建进程

需要启动大量子进程时，可以使用进程池

```py
multiprocessing.Pool([process,])
```

一般只指定`process`参数

#### 进程池的常用方法

`apply(func[,args[,kwds]])`

`apply_async(func[,args[,kwds[,callback[,eror_callback]]]])`

`close()`阻止后续任务提交到进程池，当所有任务执行完毕后，工作进程会退出

`terminate()`不必等待未完成的任务，立即停止工作进程

`join()`阻塞主进程，等待工作进程结束，调用join()前必须先调用` close()或者terminate()`

```py
import os
import multiprocessing
import time


def func():
    for i in range(2):
        print(os.getpid(), i)
        time.sleep(0.5)


if __name__ == "__main__":
    process_num = 5
    pool = multiprocessing.Pool(process_num)
    for p in range(process_num):
        # pool.apply(func)#同步
        pool.apply_async(func) #异步
    pool.close()
    pool.join()
    print(f"当前进程名字{multiprocessing.current_process().name}:end")
```

### 进程间通信

进程之间不共享全局变量

#### 使用`Queue`通信

通过使用`multiprocessing.Queue()`实现进程间的通信

```py
from random import randint
import time
import multiprocessing
import os

def func1(queue):
    while True:
        num = randint(1,50)
        queue.put(num)
        print(f"进程{os.getpid()}向队列放入{num}")
        time.sleep(0.5)

def func2(queue):
    while True:
        num = queue.get()
        print(f"从队列{os.getpid()}中取出了{num}")
        time.sleep(0.5)


if __name__ == "__main__":
    queue = multiprocessing.Queue()
    p1 = multiprocessing.Process(target=func1,args=(queue,))
    p2 = multiprocessing.Process(target=func2,args=(queue,))

    p1.start()
    p2.start()
    p1.join()
    p2.join()
```

`mutiprocessing.Queue`可能会有兼容性问题,如果进程池要使用，可以使用

`multiprocessing.Manager().Queue()`

```py
if __name__ == "__main__":
    queue = multiprocessing.Manager().Queue()
    pool = multiprocessing.Pool()
    pool.apply_async(func1,args=(queue,))
    pool.apply_async(func2,args=(queue,))
    pool.close()
    pool.join()
```

## 多线程

线程是处理器任务调度和执行的基本单位

一个进程至少有一个线程，也可以有多个线程

多个线程之间可以共享数据

线程运行出错后如果没有捕获，则会导致整个进程崩溃

多线程是指在同一进程中执行多个任务

### 使用`threading.Thread`创建线程

`group`:应当始终为None，保留给将来实现`ThreadGroup`类扩展使用

`target`:由`run()`方法发起调用的可调用对象，默认为None

`name`:线程名称，默认为None则会以"Thread-N"的形式构造唯一名称,其中N为一个较小的十进制数值，或者是"Thread-N(target)"的形式

`args`:针对目标调用的参数元组

`kwargs`:针对目标调用的关键字参数字典

`daemon`:是否为守护线程,True或者False,默认为None则继承父进程

#### 常用方法

`start()`启动线程，调用线程的`run()`方法

`run()`定义线程的行为，默认调用传入的target对象

`join([timeout=None])`阻塞主线程，直到当前线程运行完成或达到timeout时间

`threading.enumerate()`查看都有哪些线程

`threading.current_thread()`返回当前线程实例

```py
import threading
import time

def func():
    flag = 0
    while True:
        print(f"线程{threading.current_thread().name}", f"{flag}" * 5)
        flag = flag ^ 1
        time.sleep(0.5)

if __name__ == "__main__":
    t1 = threading.Thread(target=func)
    t2 = threading.Thread(target=func)
    t1.start()
    t2.start()
```

### 自定义Thread子类

## 互斥锁

### 线程安全问题

某个线程要更改共享数据时，先将其锁定，此时其他线程不能更改。直到该线程释放资源，将资源的状态变为“非锁定”，其他的线程才能再次锁定该资源

保证了每次只一个线程进行写入操作，从而保证了多线程情况下数据的正确性

#### 线程锁的使用

使用`thrading.Lock()`创建互斥锁

使用`lock.acquire([blocking=True][,timeout=-1])`来获得锁

使用lock.release()释放锁

```py
import time
import threading

def func():
    global g_num
    for _ in range(10):
        lock.acquire()
        temp = g_num + 1
        time.sleep(0.01)
        g_num = temp
        print(f"当前线程{threading.current_thread().name}",g_num)
        lock.release()

if __name__ == "__main__":
    g_num = 0
    lock= threading.Lock()
    Threads = [threading.Thread(target=func,name="线程"+str(i)) for i in range(1,4)]
    [t.start() for t in Threads]
    [t.join() for t in Threads]    
```

## 卖票案例

```py
import time
import threading


def sale_ticket():
    global ticket_num
    while True:
        lock.acquire()
        if ticket_num <= 0:
            #释放
            lock.release()
            break
        time.sleep(0.01)
        ticket_num -= 1    
        print(f"当前{threading.current_thread().name}卖了一张票", f'还剩{ticket_num}')
        lock.release()


if __name__ == "__main__":
    ticket_num = 100
    lock = threading.Lock()
    Threads = [threading.Thread(target=sale_ticket, name="窗口" + str(i)) for i in range(1, 4)]
    [t.start() for t in Threads]
    [t.join() for t in Threads]
```

## GIL

Python 全局解释器锁,同一时间只允许一个线程保持Python解释器的控制权，这意味着任何时间点都只能有一个线程处于执行状态 

## 进程线程对比

适合使用多线程的情况:

- I/O 密集型任务  :如网络请求、文件读写等

- 对资源共享要求高

适合使用多进程的情况:

- CPU 密集型任务:如进行复杂的科学计算、数据处理等人物

- 需要隔离的任务
